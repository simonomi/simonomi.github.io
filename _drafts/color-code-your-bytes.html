this visual pattern recognition can be especially helpful when scrolling through a large binary file.

demonstration of colors showing where you are in a file

if you’ve used a hex editor with color-coding before, you may have noticed something different about the way i’m choosing to color-code bytes.

most colorful hex editors have a few categories that they use to color each byte. categories like 00 bytes, printable ASCII, ASCII whitespace, other ASCII, non-ASCII, or FF bytes.

these color categories are enough to pick out common patterns like repeated null bytes and ASCII strings. there’s also enough variation to easily track scrolling.

example of picking a string out of a file

i, however, am going further, with 18 total categories: one for each leading nybble (0X, 1X, 2X...) and two extras for 00 andFF.

having more colors makes it easy to recognize even more complex patterns. try for yourself!

- indices
- header
- compressed
- binary
- animations

so what can you do about it?

  • switch to hexyl (or others?)
  • if you make a hex editor, add an option
